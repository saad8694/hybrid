# 02. Architecture

## Overview
The framework follows a modular architecture with separate layers for web, API, mobile, database, and shared infrastructure.

## Layers
- Web layer for UI automation and page objects
- API layer for reusable REST clients
- Mobile layer for emulation-based mobile web testing
- Data layer for database operations
- Framework layer for shared utilities, reporting, logging, and base classes

## Design Goals
- Clear separation of concerns
- Reusable components
- CI-friendly execution
- Easy extension to future platforms
