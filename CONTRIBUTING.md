# Contributing

## Workflow
1. Fork or clone the repository.
2. Create a branch from `develop`.
3. Make focused changes with clear commit messages.
4. Run the relevant tests locally.
5. Open a pull request with context and results.

## Coding standards
- Use TypeScript for framework code.
- Prefer reusable helpers over duplicate logic.
- Keep page objects and API clients structured and reusable.
- Add or update tests when changing behavior.

## Commit message tips
- Use short, descriptive messages such as:
  - `feat: add API client layer`
  - `fix: stabilize login page selectors`
  - `docs: update contribution guide`

## Pull request checklist
- Tests pass locally
- CI workflow succeeds
- Documentation updated when needed
- No sensitive data committed
